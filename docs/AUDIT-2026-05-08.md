## 1. Project Overview

**vidaenlosextremos-vr** is an A-Frame/Three.js interactive 3D simulator for exploring deep-sea habitats. It presents a photogrammetry-based ROV (Remotely Operated Vehicle) mission control interface with telemetry HUD, waypoint system, and multi-input support.

### Tech Stack
| Layer | Technology |
|-------|-----------|
| 3D Engine | A-Frame 1.4.2 / Three.js (r137) |
| Language | Vanilla JavaScript (ES6, no bundler) |
| Styling | CSS3 with custom properties, Glassmorphism |
| Data | JSON (dives, waypoints, locales, version) |
| Assets | GLTF/GLB 3D models, JPEG/PNG textures, photos |

### Missions
| Key | Site ID | Name | Depth |
|-----|---------|------|-------|
| `whale-fall` | S0883 | The Whale Fall | 3,895m |
| `clam-bed` | S0889 | Vesicomyidae Clam Bed | 618m |
| `giant-corals` | S0892 | Giant Corals (*Paragorgia sp.*) | 483m |

### Data Flow
```
index.html ──(click card)──→ habitat.html?site=whale_fall
                                  │
                    loader.js ────┤
                    ├─ fetch dives.json → config, spawn, model path
                    ├─ fetch waypoints.json → spawn POIs
                    ├─ fetch icons.html → inject SVG sprite
                    ├─ fetch version.json → console log + UI display
                    └─ setAttribute('gltf-model') → model-loaded event
                                  │
                    rov-model-handler.js
                    ├─ apply textures (materialXX_diffuse.jpeg)
                    ├─ auto-scale to 20m target
                    ├─ compute floor limit
                    └─ trigger rov-visual-blender.js
                       ├─ generateSilhouetteMask (512x512)
                       ├─ applyMaskFade (shader injection)
                       └─ generateHeightmap (128x128)
                                  │
                    rov-main.js (game loop @ 60fps)
                    ├─ updateGamepad()
                    ├─ ROV.updateKeyboard()
                    ├─ touch virtual controls
                    ├─ [every 4th frame]:
                    │   ├─ ROV.waypoints.update() → proximity check
                    │   ├─ heading / depth / coords HUD update
                    └─ ROV.physics.applyMove() → terrain collision
```

### Input Pipeline
```
Keyboard ──→ keyState{} ──→ ROV.updateKeyboard() ──→ ROV.physics.applyMove()
                                                     ──→ ROV.actions.look()
Gamepad  ──→ processGamepadInput() ──→ ROV.physics.applyMove()
                                     ──→ handleGamepadRotation() → ROV.actions.look()
Touch    ──→ hold buttons (activeAction) ──→ ROV.physics.applyMove()
        ──→ drag (dragData) ──→ applyRotation() → ROV.actions.look()
```

### 3.4 Systems JS (8 files, 790 lines)

#### `localization.js` (70 lines)
- `init()` — detect language from localStorage → browser → default `es`
- `loadLocales()` — fetch JSON with path detection for nested deployments
- `setLanguage(lang)` — swap locale, save to localStorage, dispatch `languageChanged` event
- `t(key)` — dot-notation path resolution with fallback to key
- `updateDOM()` — iterate `[data-i18n]` elements, handle `data-i18n-attr` for attributes

#### `rov-physics.js` (92 lines)
- `applyMove(surge, sway, heave, isTurbo)` — vector movement relative to camera heading
  - Speed formula: `baseMoveSpeed * (fov/80) * speedLevel * turbo`
  - Total angle = rig Y + camera Y (both in radians)
  - Surge/Sway resolved via sin/cos of total angle
- **Terrain-following:** samples `ROV.config.heightmap` at current XZ, clamps Y to floor
  - Falls back through: heightmap → floorLimitOverride → floorLimit → startPosition.y
- `updateSpeedUI()` — updates speed display text

#### `rov-actions.js` (115 lines)
- `cycleHUD()` — 3-mode toggle: Full → Telemetry → Cinematic → Full
- `resetPosition()` — reset rig+rotation to spawn, reset pivot+cam rotation
- `toggleFullscreen()` — browser fullscreen API
- `toggleGyro()` — iOS permission flow + A-Frame magicWindowTrackingEnabled
- `changeSpeed(direction)` — ±1 through speedLevels array
- `scanWaypoint()` — retrieve waypoint data → open modal → mark visited → flash button
- `look(yawDelta, pitchDelta)` — centralized rotation: yaw→rig.Y, pitch→pivot.X (clamped ±80°)

#### `rov-model-handler.js` (92 lines)
- **Named constants:** `MODEL_Y_OFFSET: -2`, `MODEL_Z_OFFSET: -10`, `TARGET_SIZE: 20`
- `setupModel(mesh)`:
  1. **Texture loading:** pattern `materialXX_diffuse.jpeg` via THREE.TextureLoader, applied with `DoubleSide`
  2. **Auto-scale:** bounding box → scale factor to reach TARGET_SIZE
  3. **Position:** center model then apply Y/Z offsets
  4. **Floor:** compute bottom of geometry as `floorLimit`
  5. **Extended floor:** set color, position, animate opacity
  6. **Visual blender:** trigger silhouette mask + heightmap generation
  7. **Dynamic speed:** `0.02 + log10(maxDim+1) * 0.03` — larger models move faster

#### `rov-visual-blender.js` (301 lines)
The most complex module. GPU-accelerated edge fade system.
- `generateSilhouetteMask(mesh, resolution=512, blurPasses=4, blurRadius=6, padding=0.10)`:
  - **Pass 1:** traverse all mesh vertices in world space → compute XZ bounds
  - **Pass 2:** project triangles using index buffer → draw white on black canvas
  - **Solidify:** threshold pass to close sub-pixel gaps
  - **Blur:** multi-pass box blur (horizontal then vertical)
  - Returns `{ texture: CanvasTexture, bounds: {minX, maxX, minZ, maxZ} }`
- `generateHeightmap(mesh, resolution=128, hoverOffset=0.15)`:
  - Projects vertices to XZ grid, stores maximum Y per cell
  - Fills gaps with global minimum Y
  - Adds hover offset for terrain-following
  - Returns `{ data: Float32Array, resolution, bounds, minY, maxY }`
- `applyMaskFade(mesh, maskData)`:
  - Injects GLSL via `material.onBeforeCompile`
  - Vertex shader: passes world position
  - Fragment shader: samples mask texture, applies `pow(alpha, 5.0)` steepening
- `_boxBlur(ctx, w, h, radius, passes)` — private multi-pass blur

#### `rov-waypoints.js` (248 lines)
- `init()` — fetch waypoints.json, spawn waypoints, create mission HUD
- `createMissionHUD()` — injects counter into top-right-ctrls
- `spawn(data)` — create A-Frame entities: sphere + beam + sonar ring with animations
- `update()` — proximity check (minDistance: 2.0m), transform sphere→octahedron on near, track active
- `markAsVisited(id)` — green color, octahedron rotation, increment counter, check completion
- `missionComplete()` — green border, glow, completion message (idempotent via guard)
- `toggleGamifiedMode(enabled)` — show/hide HUD and waypoint helpers
- `syncUI(activeId)` — show/hide scan button and explore label

#### `rov-modal.js` (133 lines)
- `open(waypointData)` — set `isLogbookOpen`, render content by type
- `close()` — delayed content clear for CSS transition, release input lock
- Content builders: `_buildYoutube()`, `_buildGallery()`, `_buildChart()`
- `updateGalleryFocus(src, caption, thumbEl)` — crossfade gallery images

#### `rov-controls-ui.js` (151 lines)
- `init()` — event listeners + back button handler
- `setupBackHandler()` — `popstate` listener for Android back (modal → controls → menu)
- `pushUIHistory(layer)` — history API state management
- `bindPress()` — deduplication wrapper (max 250ms between pointerup+click)
- `toggle(show)` — controls overlay with tab detection (gamepad if connected, else keyboard)
- `toggleMenu(show)` — system menu sidebar
- `switchTab(tabName)` — gamepad/keyboard view + hint toggling
- `updateGPIcons()` — convert text labels (BTN Y, BTN B) to SVG icons
- `checkGPExit(gp)` — B=close overlay, Select=switch tab, Start=menu

### 3.5 Input JS (2 files, 248 lines)

#### `rov-input-keyboard.js` (73 lines)
- `keyState{}` — physical key polling via `e.code`
- `keydown` handler: Escape (layered: modal→controls→menu→open), Enter/H/R/1/2 actions
- `ROV.updateKeyboard()` — called every frame
  - Blocks when `<input>` focused or modal open
  - WASD → surge/sway, Space/Shift → heave, E/Q → zoom, Arrows → camera rotation

#### `rov-controls.js` (175 lines)
- `updateGamepad()` — iterate gamepads, call `processGamepadInput()`
- `processGamepadInput(gp)`:
  - LS → surge/sway with 0.1 deadzone
  - L1/R1 → ascend/descend
  - RS → rotation with vertical handicap (0.65) for single-axis preference
  - L2/R2 → zoom
  - D-pad ↑↓ → speed
  - Square → HUD, Cross → scan, Select → reset
- `initUIControls()` — bind click/hold for all touch buttons
- `initRotationControls()` — shared drag state (mouse + touch)
  - Interactive target detection prevents rotation over UI elements
  - Sensitivity differs: touch=0.15, mouse=0.2
  - Single-axis preference: horizontal or vertical (not both)

### 3.6 Orchestration (2 files, 172 lines)

#### `rov-main.js` (69 lines)

- `initSystem()` — init rotation controls, disable A-Frame native look, init localization, init waypoints, start loop
- `updateLoop()` — 60fps game loop:
  - Every frame: gamepad, keyboard, touch movement
  - Every 4th frame: waypoints proximity, heading/depth/coords HUD update
- Depth formula: `baseDepth - pos.y` (higher Y = shallower displayed depth)
- Gamepad connect event → debug console notification

## 5. What Was Not Changed

### Intentionally Preserved
- **A-Frame CDN dependency** — no bundler. Adding offline A-Frame would add ~2MB to repo.
- **Shader injection via `onBeforeCompile`** — fragile but works with A-Frame 1.4.2 / Three.js r137. Would need rewrite for newer Three.js.
- **`getAttribute()` for DOM reads** — slower than `object3D` direct access, but comments in `rov-main.js` acknowledge this as a known optimization.
- **English locale typo** — "Maria Emilia Bravo" (missing accent on María) — minor, user-facing content.

### Known Technical Debt (Documented)
1. Shader injection pattern (`onBeforeCompile`) is version-sensitive — a Three.js upgrade would break the silhouette mask
2. No module bundler — 15 separate `<script>` tags in dependency order
3. No automated tests
4. Static A-Frame version pinned (1.4.2)
5. No service worker / offline support
